## EMAIL SYSTEEM TECHNISCHE SPECIFICATIES

### 1. DATABASE STRUCTUUR
Het emailsysteem maakt gebruik van drie database tabellen:

#### 1.1 email_config
Bevat de SMTP configuratie instellingen:
- id (INT, AUTO_INCREMENT, PK)
- provider (ENUM: 'brevo', 'sender_net', 'aws_ses')
- smtp_host (VARCHAR)
- smtp_port (INT)
- smtp_user (TEXT, versleuteld)
- smtp_pass (TEXT, versleuteld)
- from_email (VARCHAR)
- from_name (VARCHAR)
- is_active (BOOLEAN)
- created_at (TIMESTAMP)
- updated_at (TIMESTAMP)

#### 1.2 email_templates
Bevat de email template configuratie:
- id (INT, AUTO_INCREMENT, PK)
- type (ENUM: 'tenant_welcome', 'admin_invite', 'guest_confirmation', 'guest_password_reset', 'marketing')
- subject (VARCHAR)
- content (TEXT)
- text_content (TEXT, optioneel)
- tenant_id (INT, FK naar tenants.id)
- language_code (VARCHAR)
- is_default (BOOLEAN)
- created_at (TIMESTAMP)
- updated_at (TIMESTAMP)

#### 1.3 email_log
Bevat de email verzend log:
- id (INT, AUTO_INCREMENT, PK)
- tenant_id (INT)
- user_id (INT, optioneel)
- recipient_email (VARCHAR)
- subject (VARCHAR)
- template_type (VARCHAR)
- template_id (INT)
- provider (VARCHAR)
- status (ENUM: 'pending', 'sent', 'failed')
- error_message (TEXT)
- sent_at (TIMESTAMP)
- created_at (TIMESTAMP)

### 2. EMAIL TEMPLATE SYSTEEM

#### 2.1 Template Types
Er zijn vijf template types beschikbaar:
1. tenant_welcome - Voor nieuwe tenant aanmeldingen
2. admin_invite - Voor admin uitnodigingen
3. guest_confirmation - Voor gast registratie bevestigingen
4. guest_password_reset - Voor gast wachtwoord reset
5. marketing - Voor marketing campagnes

#### 2.2 Template Variabelen
Templates maken gebruik van Mustache-achtige placeholders:
- {{tenant_name}} - Tenant naam
- {{user_name}} - Gebruikersnaam
- {{password_reset_link}} - Wachtwoord reset link
- {{invitation_link}} - Uitnodigingslink
- {{verification_code}} - Verificatie code
- {{campaign_name}} - Campagne naam
- {{campaign_message}} - Campagne bericht
- {{action_url}} - Actie URL
- {{action_text}} - Actie tekst
- {{unsubscribe_url}} - Afmeld URL

## GATED ONBOARDING (KYC-LIGHT) — ACCOUNT STATUS SYSTEEM

### STATE MACHINE
```
unverified ──(barman valideert ID)──> active
unverified ──(barman weigert)──────> unverified (met cooldown)
active ──────(admin blokkeert)──────> suspended
suspended ───(admin deblokkeert)────> active
```

### GAST RECHTEN PER ACCOUNT STATUS

| Actie | unverified | active | suspended |
|-------|:----------:|:------:|:---------:|
| Inloggen | ✅ | ✅ | ❌ |
| Dashboard bekijken | ✅ | ✅ | ❌ |
| QR code tonen | ✅ (met "UNVERIFIED" badge) | ✅ | ❌ |
| Saldo storten | ❌ | ✅ | ❌ |
| Betalen aan bar | ❌ | ✅ | ❌ |

### TOEGANGSRECHTEN PER ROL
- Superadmin: Platform beheerder (geen account_status restricties)
- Admin: Kan gasten blokkeren (`suspend`) en deblokkeren (`unsuspend`) via `/api/admin/suspend_user`
- Bartender: Kan unverified gasten verifieren via `/api/pos/verify` (met birthdate match)
- Gast unverified: Kan inloggen, dashboard bekijken, QR tonen. Kan NIET storten of betalen.
- Gast active: Volledige toegang (bestaand gedrag).
- Gast suspended: Kan niet inloggen (login wordt stil geblokkeerd, geen info leakage).

### RATE LIMITING (configureerbaar per tenant)
- `verification_soft_limit` (default 15): Waarschuwing per barman per uur
- `verification_hard_limit` (default 30): Absolute blokkade per barman per uur
- `verification_cooldown_sec` (default 180): Seconden wachttijd na mismatch
- `verification_max_attempts` (default 2): Max verificatiepogingen per gast per 24u
